## MODIFIED Requirements

### Requirement: Generated artifacts stay on the codegen side of the boundary

Generated artifacts MUST stay device-scoped, typed, and free of Alloy-owned public HAL surface.

#### Scenario: Generated bring-up code remains inside the contract boundary
- **WHEN** startup or system clock bring-up code is emitted
- **THEN** it is published under `generated/devices/<device>/startup.cpp` or
  `generated/runtime/devices/<device>/system_clock.hpp`
- **AND** it does not emit `namespace alloy`
- **AND** it does not emit public HAL driver classes
- **AND** it does not implement board selection or ownership models

#### Scenario: Alloy no longer owns handwritten foundational startup translation units
- **WHEN** foundational families are published under the active contract
- **THEN** the low-level reset-handler/vector-table translation unit is generated by codegen
- **AND** Alloy consumes the generated startup artifact instead of handwritten per-device startup
