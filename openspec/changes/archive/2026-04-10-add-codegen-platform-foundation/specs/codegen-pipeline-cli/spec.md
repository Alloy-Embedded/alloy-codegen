## ADDED Requirements

### Requirement: Stage-Oriented CLI
The system SHALL expose a CLI that mirrors the architecture stages and supports scoped
execution.

#### Scenario: CLI exposes explicit pipeline stages
- **WHEN** a user runs the CLI help or invokes a stage command
- **THEN** the CLI SHALL expose at least the stages:
  `fetch`, `patch`, `normalize`, `validate`, `emit`, `publish`, and `pipeline`

#### Scenario: CLI supports scoped execution
- **WHEN** a user targets one vendor, family, or device
- **THEN** the CLI SHALL support executing the requested stage only for that scope
- **AND** it SHALL avoid requiring whole-repo regeneration for narrow development loops

### Requirement: Structured CLI Output
The CLI SHALL produce machine-readable outputs for CI and automation.

#### Scenario: CLI can emit structured status
- **WHEN** a stage completes in CI or scripted automation
- **THEN** the CLI SHALL be able to emit structured status and result data
- **AND** the output SHALL include scope, success/failure, and references to manifests or
  reports generated by the stage
